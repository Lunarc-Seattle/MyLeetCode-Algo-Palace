## 👋 About This Repository

This repository records my journey of improving algorithmic thinking for software engineering interviews.

Rather than only collecting accepted solutions, I focus on:

- identifying reusable problem-solving patterns
- writing clean and readable code
- analyzing time and space complexity
- improving abstraction and modeling skills
- turning repeated mistakes into reusable templates

> **Core belief:**  
> Algorithms are not just tricks. They are structured ways of thinking.

---

### 🏮 Manifesto 

* **Logic is the New Zen:** In this palace, every semicolon is a brick 🪨, and every algorithm is a ritual.
    
* **Time & Space Complexity:** We don't just solve problems; we hunt for the optimal $O(1)$ in a $O(n!)$ world.
